# AGENTS.md

## 目的
このリポジトリは、VRChatアバターを「おちびちゃんズ」向けに変換する Unity Editor 拡張（個人開発）です。  
レビュー/実装では **既存挙動を壊さない小差分** を最優先にしてください。回答は日本語。

## スコープ境界
- `Assets/Aramaa/OchibiChansConverterTool/`: 配布ソース
- `docs/`: ユーザー向け文書
- `Tools/release/`: リリース運用

無関係なタスクで境界をまたいだ変更をしない。

## 技術前提
- Unity 2022.3 系 Editor拡張
- VRCSDK3A / Modular Avatar 連携あり
- SDK/MA未導入環境で壊れない分岐を維持
- `SerializedObject` / `Undo` / `AssetDatabase` / `PrefabUtility` / `Selection` は回帰リスク高として扱う

## 変更時の最重要ポイント
- 変換順序、複製→適用、Undo単位、Blueprint ID、Prefab候補優先順位、FaceMeshキャッシュ互換、文言キー、MA分岐を崩さない
- 新機能や仕様変更を勝手に入れない
- 無関係なリファクタ・広域整理をしない

## レビュー方針
### 指摘する
- バグ/回帰リスク、null/参照切れ/条件漏れ、誤アセット操作
- Unity Editor APIの危険な使い方
- ローカライズ・ログ整合崩れ
- 依存なし環境で壊れる変更

### 原則指摘しない
- 大規模開発前提の抽象化、DI先回り導入
- 全面リネーム/全面再編
- 実害の薄いスタイル論争

### レビュー出力
1. 必須で直すべき点
2. 余裕があれば直したい点
3. 今回は見送ってよい点

各項目は「問題 / 理由 / 最小修正方針」を簡潔に。重大問題がなければ「今回は重大な問題なし」。

## ローカライズ/キャッシュ
- ベタ書き文字列追加は最小限、既存ローカライズ経路を優先
- Library配下キャッシュはVCSに含めない
- キャッシュ形式/キー変更時は互換影響を明示

## コスト最適化
- 先に結論を短く示し、詳細は必要時のみ
- 長文・重複・タスク外提案を避ける

## 競合時優先順位
1. ユーザーの明示指示
2. この AGENTS.md
3. 既存コード慣習
